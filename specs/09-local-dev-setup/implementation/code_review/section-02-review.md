# Code Review: section-02-test-integration-sh

## Issues Found

### 1. Dead `jq` dependency check (confidence: 95) — Auto-fixed
`jq` is no longer used in the script after removing the extraction block, but `check_cmd jq` still blocks execution if `jq` is missing. Removed the line.

### 2. No required-var check after sourcing (confidence: 88) — Auto-fixed
No verification that sourcing `.env.local` actually set required variables. A stale `.env.local` (from an older version of `local-dev.sh`) would silently pass sourcing and fail at runtime. Added a sentinel check for `SMGR_API_URL` with an actionable error message pointing to the regenerate command.

### 3. `set -euo pipefail` + sourcing user-edited file — Accepted
If `.env.local` contains a line that references an unbound variable, `set -u` will kill the script silently. Since `.env.local` is machine-generated by `local-dev.sh print_setup_env_vars` (not hand-edited), this is an acceptable risk. Full coverage is provided by `verify.sh`.
