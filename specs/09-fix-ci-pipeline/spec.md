## Root Causes

### Failure 1: `search_events` RPC returns empty — assertion is silently skipped

**File:** `web/__tests__/integration/tenant-isolation.test.ts:195-202`

```typescript
it("should return only Alice's events when Alice calls search_events", async () => {
  const { data } = await aliceClient.rpc("search_events", {
    p_user_id: aliceId,
    query_text: "Test enrichment",
  });
  if (data && data.length > 0) {  // ← BUG: silently passes with empty results
    expect(data.every((r) => r.user_id === aliceId)).toBe(true);
  }
});
```

The `search_events` SQL function joins `enrichments` on `events` and filters `en.fts @@ plainto_tsquery(...)`. The seed layer (`setup.ts:189`) inserts enrichments with `description: "Test enrichment for {eventId}"`, so FTS *should* match `"Test enrichment"`.

**Likely cause:** The RPC is called via Alice's user client (anon key + auth token). RLS on `enrichments` may restrict the join, or the `search_events` function may not have EXECUTE granted to `authenticated`. Need to check grants.

**Fix:** Remove the `if` guard. The test must assert `data.length > 0` — if search returns empty, the test should fail, not silently pass. Then debug why results are empty (RLS on enrichments table or missing EXECUTE grant).

### Failure 2: Event type assertion expects `"photo"` but seed uses `"create"`

**File:** `web/__tests__/integration/tenant-isolation.test.ts:284`

```typescript
expect(original!.type).toBe("photo");  // ← BUG: seedUserData uses type: "create"
```

`seedUserData` (setup.ts:175) creates events with `type: "create"`. The test verifies that after a rejected UPDATE, the event's `type` is unchanged — but it compares against the wrong expected value.

**Fix:** Change `"photo"` to `"create"` on line 284.

### Failure 3: E2E `beforeAll` timeout

**File:** `web/e2e/agent.spec.ts:69`

The `beforeAll` hook signs up a test user, then calls `getConfirmationLink()` which retries up to 10 times with exponential backoff (max ~50s). The default Playwright `beforeAll` timeout is 30s.

**Contributing factor:** The `.env.local` generated by CI (ci.yml:173-177) has leading whitespace in every line due to YAML heredoc indentation:
```yaml
cat > .env.local <<EOF
          # Supabase (local test instance)
          NEXT_PUBLIC_SUPABASE_URL=...
```
This produces `          NEXT_PUBLIC_SUPABASE_URL=...` which Next.js won't parse, so the app has no Supabase connection, and signup/confirmation never works.

**Fix (two parts):**
1. Fix `.env.local` generation in `ci.yml` — remove leading whitespace from heredoc lines.
2. Increase `beforeAll` timeout to 60s (or add explicit timeout parameter) to handle Mailpit latency.

### Warning: Dangling handle prevents clean exit

After integration tests complete, Vitest reports: `"Tests closed successfully but something prevents the main process from exiting"`. This is likely an unclosed Supabase realtime subscription or HTTP client in `setup.ts`.

**Fix:** Call `admin.removeAllChannels()` or destroy the client in `afterAll` / global teardown.
